Movie API for the myFlix app

The goal is to build the server-side component of a movies web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Movie API Documentation

Request URL HTTP Method Request Body Data Format Response Body Data Format
Get a list of all movies /movies GET none A JSON object holding an array with all movies.
Get data about a single movie by title /movies/:title GET none A JSON object holding data about a single movie, containing description, genre, director, image URL, and if it is featured. Here is an example:

{ "genre": { "name": "Sci-Fi", "description": "A genre dealing with imaginative and futuristic concepts such as advanced science and technology, space exploration, time travel, and extraterrestrial life." }, "director": { "name": "George Lucas", "bio": "George Walton Lucas Jr. is an American film director, producer, and screenwriter. He is best known for creating the Star Wars and Indiana Jones franchises.", "birthyear": 1944 }, "_id": "65465b7c3d2db11d72639439", "title": "Star Wars", "description": "A young farm boy joins a group of rebels to fight an oppressive empire, discovering his connection to a mystical force known as the Force.", "featured": true }
Get data about a genre by title /movies/genre/:genreName GET none A JSON object holding data about a genre containing the properties for name and description. Here is an example:

{ "name": "Sci-Fi", "description": "A genre dealing with imaginative and futuristic concepts such as advanced science and technology, space exploration, time travel, and extraterrestrial life." }
Get data about a director by name /movies/director/:directorName GET none A JSON object holding data about a director containing the properties for name, bio and birth. Here is an example:

{ "name": "James Cameron", "bio": "This is a new bio!", "birthyear": 1954 }
Get a list of all users /users GET none A JSON object holding an array with all users.
Get data about a single user by name /users/:username GET none A JSON object holding data about the user that was added, including an ID:

{ "_id": "65465ffb3d2db11d72639443", "username": "Amanda Martin", "password": "hgrk7342!", "email": "amanda.martin@test.com", "birthday": "1990-02-01T00:00:00.000Z", "favoriteMovies": [ "65465bdd3d2db11d7263943a" ] }
Add a new user /users POST A JSON object holding data about the user to add, structured like:

{ "username": "Jane Doe", "password": "jane12345!", "email": "jane.doe@test.com", "birthday": "1987-03-08" } 		A JSON object holding data about the user that was added, including an ID:

{ "username": "Jane Doe", "password": "jane12345!", "email": "jane.doe@test.com", "favoriteMovies": [], "_id": "655024e6172f78f5f27c3cbe", "__v": 0 }
Update user username /users/:username PUT A JSON object holding data about the user to update, structured like:

{ "username": "Jane Doe", "password": "doe7890=", "email": "jane.doe@example.com" } 		A JSON object holding data about the user that was updated, including an ID:

{ "_id": "655024e6172f78f5f27c3cbe", "username": "Jane Doe", "password": "doe7890=", "email": "jane.doe@example.com", "favoriteMovies": [], "__v": 0 }
Add a movie to user favorite list /users/:username/movies/:movieID POST none A JSON object holding data about the user that was updated, including an ID:

{ "_id": "65465ffb3d2db11d72639445", "username": "Christopher Lee", "password": "mcbl2348?", "email": "christopher.lee@test.com", "birthday": "2002-06-05T00:00:00.000Z", "favoriteMovies": [ "65465c3a3d2db11d7263943d", "65465c3a3d2db11d7263943c" ] }
Remove a movie from user favorite list /users/:username/movies/:movieID DELETE none A JSON object holding data about the user that was updated, including an ID:

{ "_id": "65465ffb3d2db11d72639445", "username": "Christopher Lee", "password": "mcbl2348?", "email": "christopher.lee@test.com", "birthday": "2002-06-05T00:00:00.000Z", "favoriteMovies": [ "65465c3a3d2db11d7263943d" ] }
Delete user registration /users/:username DELETE none A text message indicating if a user was successfully removed.
